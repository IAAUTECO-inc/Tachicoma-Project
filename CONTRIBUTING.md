# Contributing to Project Tachikoma

As a sovereign infrastructure project, we maintain high standards for code integrity and legal compliance.

## Developer Certificate of Origin (DCO)
To comply with OpenChain standards, all contributions must include a "Signed-off-by" line in the commit message, certifying that you have the right to submit the code under the project's license.

`Signed-off-by: Jane Doe <jane.doe@example.com>`

## Code Standards
- **C++/Go:** All code must pass the internal security audit and static analysis.
- **Documentation:** Any new feature must include its corresponding entry in the AEGIS Traceability Matrix.
- **Privacy:** No contribution shall introduce external telemetry or non-sovereign dependencies.
